# Security Policy

## サポート対象

現在 YUWA は **開発フェーズ** にあり、本番ローンチ前です。
ローンチ後は最新版（`main` ブランチ）のみがサポート対象となります。

## 脆弱性の報告

YUWA のセキュリティ脆弱性を発見した場合は、**公開 Issue として報告せず**、以下の方法で直接ご連絡ください:

### 連絡先

- 📧 PO 青木に直接メール（メールアドレスは別途共有）
- 💬 Slack / LINE で直接 DM

### 報告に含めてほしい内容

- 脆弱性の概要
- 再現手順
- 影響範囲（どんな情報が漏れる / 改ざんされる等）
- 想定されるリスクの深刻度
- 可能であれば修正案

## 対応フロー

1. **24-48 時間以内に受信確認** をお返しします
2. 影響度の評価（**high / medium / low**）
3. 修正の優先度を決定
4. 修正実装・本番反映
5. 報告者への対応完了通知
6. 必要に応じてユーザーへの開示（GDPR / 個人情報保護法の要件に応じて）

## 開示方針

- 重大な脆弱性については、修正完了後に開示します（修正前の公開は影響を拡大するため避けます）
- 報告者は、希望があればクレジット表記をリリースノートに含めます

## 特に注意している領域

YUWA の特性上、以下の領域のセキュリティを特に重視します:

- **匿名性の担保**: 匿名投稿の投稿者特定経路が運営・第三者から発生しないこと
- **個人情報の保護**: メンバーのメールアドレス・連絡先・出欠記録の漏洩防止
- **認証 / 認可**: 招待コード / Google OAuth / RLS の正しい設定
- **モデレーション権限**: 一般メンバーが運営機能にアクセスできないこと
