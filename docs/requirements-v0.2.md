## 1. このドキュメントが追加するもの

v0.1 では「匿名意見箱」と「イベント掲示板」をコアにした。v0.2 では **YUWA をサークル管理アプリとして完成させる** ために以下を追加する。

| カテゴリ | 追加機能 |
|---|---|
| **A. 出欠管理** | 部会・イベントへの出欠記録、欠席理由の任意記入、出欠率の集計 |
| **B. スケジュール管理** | 部の予定一覧、繰り返し予定、メンバー個人のカレンダーへの ICS エクスポート |
| **C. 名簿管理** | メンバーの本格的なプロフィール、学年・所属・連絡先、ロール変更、卒業処理 |

各機能の優先度: **A（出欠）> B（スケジュール）> C（名簿）**
理由: A は部会の度に発生する高頻度のニーズ、B は A に紐づく、C は管理者用途で頻度低め。

## 2. ユーザーストーリー（追加分）

### 部員として
1. 次の部会の出欠を 1 タップで答えられる
2. 欠席する時、理由を任意で残せる（残さなくても OK）
3. 自分の出欠履歴を確認できる
4. 部の予定一覧を見られる、自分のカレンダーアプリに ICS で取り込める
5. 自分のプロフィール（学年・連絡先・自己紹介）を編集できる

### 運営として
6. 新しい部会・予定を作成し、出欠フォームを公開できる
7. 各予定の出欠状況を一覧で確認できる（出席 / 欠席 / 未回答）
8. 月次・学期ごとの出欠率を集計できる
9. メンバー名簿を一覧・検索でき、ロール（member / moderator / admin）を変更できる
10. 卒業・脱退メンバーを `status = left` に変更できる（投稿は匿名化して残す）

---

## 3. MVP スコープ（v0.2）

### やる
- 出欠記録（出席 / 欠席 / 未定の 3 ステート + 欠席理由テキスト）
- 予定の作成・編集・削除（運営のみ）
- 部全体の予定一覧
- 個人プロフィール編集
- 名簿一覧（運営のみ）、ロール変更
- 卒業・脱退処理

### やらない（v0.2 で）
- 出欠の自動リマインド通知（v0.3 候補）
- 繰り返し予定の複雑なルール（毎週月曜のような単純な繰り返しは OK、N 番目の月曜のような複雑なものは v0.3）
- カレンダー連携（Google Calendar 等への自動同期。ICS エクスポートのみ提供）
- 部費・会費管理（別アプリの領域）
- ファイル添付（写真・PDF 等）
- メッセージ機能（DM / グループチャット）
- 名簿の CSV エクスポート（v0.3 候補）

## 5. データモデル追加（v0.1 の §5 を継承）

### `members` テーブル（v0.1 から拡張）

v0.1 では `id / organization_id / display_name / role / joined_at / invite_code / anonymous_hash / status` のみだった。以下を追加:

| 追加カラム | 型 | 説明 |
|---|---|---|
| grade | text | 学年 (`B1`, `B2`, `B3`, `B4`, `M1`, `M2`, `OB`) |
| department | text | 所属（学科名等） |
| bio | text | 自己紹介（任意、 max 200 字） |
| contact_email | text | 連絡用メール（Google ログインのメールとは別、任意） |

### `schedules` テーブル（新規）

予定（部会・イベント・その他）。v0.1 の `events` は「外部イベントの同行者募集」なので別物として扱う。

> **設計判断**: `events` と `schedules` を統合するか別にするかは v0.2 実装時に再検討。現時点では「外部イベント = events」「部の予定 = schedules」と分ける案で進める。統合する場合は `type` カラムで区別。

| カラム | 型 | 説明 |
|---|---|---|
| id | uuid | PK |
| organization_id | uuid | FK |
| title | text | 例: 部会 / ハッカソン準備会 |
| description | text | nullable |
| starts_at | timestamptz | |
| ends_at | timestamptz | nullable |
| location | text | nullable |
| created_by | uuid | FK members.id |
| takes_attendance | bool | true = 出欠フォーム生成 / false = 告知のみ |
| recurrence_rule | text | nullable。RFC 5545 RRULE 形式（例: `FREQ=WEEKLY;BYDAY=MO`）。複雑なルールは v0.3 |
| status | text | `scheduled` / `cancelled` |
| created_at | timestamptz | |

### `attendances` テーブル（新規）

各予定への出欠記録。

| カラム | 型 | 説明 |
|---|---|---|
| schedule_id | uuid | FK |
| member_id | uuid | FK |
| status | text | `attending` / `absent` / `undecided` |
| absence_reason | text | nullable。`status = absent` の時のみ |
| responded_at | timestamptz | |
PK: (schedule_id, member_id)

### `roster_changes` テーブル（新規）

名簿の変更履歴（監査ログ）。

| カラム | 型 | 説明 |
|---|---|---|
| id | uuid | PK |
| member_id | uuid | FK |
| changed_by | uuid | FK members.id（運営） |
| change_type | text | `role_changed` / `status_changed` / `profile_updated` |
| before_value | text | JSON 文字列で変更前を保存 |
| after_value | text | JSON 文字列で変更後を保存 |
| reason | text | nullable |
| created_at | timestamptz | |

## 6. 認証フロー（変更なし）

v0.1 の招待コード + Google ログインのまま。追加で加入後にプロフィール編集を促す。

```
新メンバー Google ログイン → members レコード作成 → トップ画面
                                                    ↓
                                  「プロフィール設定しませんか？」案内（バナー）
                                                    ↓
                                            プロフィール編集画面
```

バナーはあくまで案内、必須にはしない。「あとで」できる。

---

## 7. 匿名性の仕様（変更なし）

v0.1 のハッシュ化匿名 + モデキュー方式を維持。出欠・スケジュール・名簿はすべて記名前提なので、匿名性の議論は意見箱に限定される。

---

## 8. モデレーション運用（追加）

### 名簿変更の通知
- 運営が他メンバーのロールを変更したら、変更されたメンバー本人に通知（v0.3 で実装、v0.2 では監査ログのみ）
- 運営が他メンバーを `status = left` にした場合も同様

### 出欠の改ざん防止
- 出欠は本人のみが回答・変更できる（運営も改ざん不可）
- 運営は集計のみ可能、個別出欠の編集は不可
- 「部会に行ったのに出欠忘れた」ケースは本人が事後修正

## 9. 利用規約（追加分）

v0.1 ドラフトに以下を追加:

- 個人情報の保存項目に「学年・所属・連絡先メール・自己紹介」を追記
- 出欠記録は匿名化されず記名で運営が確認できる旨を明記
- プロフィールは退会時に匿名化される（display_name は「(退会済)」表示）

詳細は [`legal/terms-of-service-v1-draft.md`](legal/terms-of-service-v1-draft.md) を参照。

## 10. 非スコープ（v0.2 でやらないこと）

- 部費・会費管理
- ファイル添付
- メッセージ機能（DM / グループチャット）
- 名簿 CSV エクスポート

## 11. マイルストーン（暫定）

v0.1 ローンチ後、Phase 5 として実装:

| Sprint | 内容 |
|---|---|
| Sprint 1 | 出欠管理（schedules + attendances テーブル + 出欠フォーム + 一覧） |
| Sprint 2 | スケジュール管理（一覧 UI + ICS エクスポート + 繰り返し予定の単純対応） |
| Sprint 3 | 名簿管理（プロフィール拡張 + 名簿一覧 + ロール変更 + 卒業処理） |

各 Sprint は 1-2 週間想定。エンジニアのペースに合わせて柔軟に。
